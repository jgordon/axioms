Predicate: enable

Summary

(enable e1 e2)

Eventuality e1 is a member of every causal complex for eventuality e2 but not the cause of e2.

Characterizing Axioms

If an eventuality e1 is required for any way of bringing about e2, then we can use the two-argument predicate enable.

(forall (e1 e2)
  (iff (enable e1 e2)
       (forall (s)
         (if (causalComplex s e2) (enable0 e1 e2 s)))))

If e1 enables e2, then (not e1) causes (not e2).

(forall (e1 e2)
  (iff (enable e1 e2)
       (forall (e3)
         (if (not' e3 e1)
             (exists (e4)
               (and (not' e4 e2) (cause e3 e4)))))))

An eventuality cannot happen before an eventuality it enables.

(forall (e1 e2)
  (if (enable e1 e2) (not (before e2 e1))))

Additional Axioms

Something being near a person is an enabling condition for perceiving it.

(forall (p x e2)
  (if (perceive' e2 p x)
      (exists (e1 s)
        (and (near' e1 x p s) (enable e1 e2)))))

When something is perceived, there is a sense organ whose intact-ness enables the perception.

(forall (p x e2)
  (if (and (person p) (perceive' e2 p x))
      (exists (o e1)
        (and (senseOrgan o p) (intact' e1 o) (enable e1 e2)))))
