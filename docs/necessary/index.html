Predicate: necessary

Summary

(necessary e c)

e is necessary with respect to a set of constraints c.

Characterizing Axioms

An eventuality is necessary if the set of constraints implies it. The predicate necessary has two arguments, an eventuality and a set of constraints.

(forall (e c)
  (if (necessary e c)
      (and (eventuality e) (eventualities c))))

An eventuality e is necessary with respect to a set of constraints c iff c implies e.

(forall (e c)
  (iff (necessary e c)
       (imply c e)))

Necessity is a positive modality because of the transitivity of imply.

(forall (e c)
  (if (necessary e c)
      (PosMod e necessary)))

All Axioms

  (iff (necessary e c)
       (imply c e)))

Necessity is a positive modality because of the transitivity of imply.

  (if (necessary e c)
      (PosMod e necessary)))

It is a theorem that if an eventuality is possible, its negation is not necessary.

(forall (e c e1)
  (if (and (possible e c) (not' e1 e))
      (not (necessary e1 c))))

If the likelihood of an eventuality is the top of the likelihood scale given constraints c, then it is entailed by c.

(forall (d e c s)
  (if (and (likelihood d e c) (likelihoodScale s))
      (iff (top d s) (necessary e c))))
